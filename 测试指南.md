## 5. 怎么测「报名」

1. 报名只需要填 UID，**格式必须是标准 UUID**（8-4-4-4-12 位十六进制）。
2. 可以直接用这几个没报过名的测试串：
   - `11111111-1111-1111-1111-111111111111`
   - `22222222-2222-2222-2222-222222222222`
   - `019e95d8-abcd-1234-5678-90abcdef0001`
3. 同一个 UID 只能报名一次，重复会提示"已报名，请登录"。
4. 报名成功后系统会从码池分配一个邀请码；**活动"进行中"时立即显示邀请码，活动"未开始"时只提示报名成功、邀请码开始后才显示**。

## 6. 怎么测三个活动阶段（未开始 / 进行中 / 已结束）

阶段由后台「活动管理」控制，默认「自动」按北京时间判断，也可手动强制。

1. 进后台 → 「活动管理」标签。
2. 「活动状态」下拉选 **自动**，然后：
   - 想看「未开始」：把开始时间填一个**未来**时间（如 `2026-12-01 10:00`），保存。
   - 想看「进行中」：开始时间填**过去**、结束时间填**未来**（如开始 `2026-07-01 10:00`、结束 `2026-08-31 00:00`），保存。
   - 想看「已结束」：把结束时间填一个**过去**时间（如 `2026-07-10 00:00`），保存。
3. 不想算时间时，「活动状态」直接选 **强制进行中 / 强制未开始 / 强制已结束**，保存即生效（忽略时间）。
4. 改完刷新活动主页 `http://localhost:3210/` 即可看到对应形态。

> 目前已按正式方案设成：开始 `2026-07-16 16:00`、结束 `2026-07-23 24:00`。所以今天(7/15)打开主页会显示「活动尚未开始」——这是正确的。想立刻预览进行中的榜单，按第 3 点选「强制进行中」。

## 10. 完整 API 接口清单（参考）

用户端（无需登录，部分需验证码）：

1. `GET /api/meta` —— 活动阶段、奖励配置、名额情况。
2. `GET /api/captcha` —— 取图形验证码（返回 svg + token）。
3. `POST /api/register` —— 提交报名，body：`{uid, agree, captchaToken, captchaAnswer}`，返回 confirmToken。
4. `POST /api/claim` —— 确认领码，body：`{confirmToken}`，返回邀请码（未开始时为 null）+ 登录令牌。
5. `POST /api/recover` —— 登录，body：`{uid, captchaToken, captchaAnswer}`，返回邀请码 + 登录令牌。
6. `POST /api/myrank` —— 查战绩，body：`{rankToken}` 或 `{uid, captchaToken, captchaAnswer}`。
7. `GET /api/leaderboard` —— 公开排行榜前 10（含每行奖品）。
8. `POST /api/winner-info` —— 提交领奖信息，body：`{rankToken, xhs, name, phone, address}`。

后台（需先 `POST /api/admin/login` 拿到会话 cookie）：

9. `GET /api/admin/overview` —— 总览数据。
10. `GET /api/admin/settings` / `POST /api/admin/settings` —— 读/写配置（奖励、时间、阶段、风控阈值等）。
11. `POST /api/admin/codes/import` —— 导入邀请码 CSV。
12. `POST /api/admin/activity/publish` —— 立即刷新榜单（发布CC推入的数据）。
13. `GET /api/admin/export` —— 导出全量数据 CSV（含领奖信息）。
14. `GET/POST /api/admin/shadows` 等 —— 激励号增删改。
15. `POST /api/admin/adjust` —— 人工修正某码的邀请人数。
16. `GET /api/admin/logs` —— 操作日志。

## 11. 重要注意事项

1. **测试库里都是假数据**（我造的测试账号、假邀请码、假激励号）。正式上线前需要清空数据库、重新灌入真实邀请码。
2. **活动开始前记得先把真实邀请码灌够**（后台「邀请码」导入，或走 CC 的 `/api/ingest/codes`）。因为开始前就允许报名占码，码不够会导致报名领不到码。
3. **口令与密钥保密**：后台口令、ingest 密钥都不要外发；上线到服务器时这些会改成环境变量注入的新值，不用当前这套。
4. **激励号占用的是真实邀请码**：它们的邀请人数由后台"自动配分/手动设置"决定，不受真实兑换数据影响；别人从榜上复制它的码去兑换是能兑的（视为增长）。
5. **冲榜奖与达标奖不可叠加**：进前十只发冲榜奖，其余发达标奖（最高一档）。
6. 数据同步（CC 每小时读飞书推数据）这部分还没做，需要等飞书表建好，所以现在榜单人数不会自己变，靠后台手动改/生成来测试。
